# 0.11.1 — 2026-06-20

## Changed

- **`secedgar_dataframe_query` invalid-SQL classification** — via mcp-ts-core ^0.10.9, a SELECT-shaped statement (`SELECT`/`WITH`) that parses but fails to prepare for a reason other than a missing table — a mistyped column, unknown function, or invalid expression — now surfaces `data.reason: 'invalid_sql'` with the DuckDB binder detail preserved in `data.binderMessage`, instead of the misleading `non_select_statement`. ([mcp-ts-core #236](https://github.com/cyanheads/mcp-ts-core/issues/236))
- **Plugin display identity unscoped** — `.claude-plugin/plugin.json`, `.codex-plugin/plugin.json`, and `.codex-plugin/mcp.json` carry the bare `secedgar-mcp-server` machine name in their `name`/server-key/`interface.displayName` fields; the `npx -y` install arg keeps the scoped `@cyanheads/secedgar-mcp-server` package name.
- **`.codex-plugin/plugin.json` `longDescription`** — populated for the 0.10.9 plugin-manifest lint, which now requires non-empty descriptions.
- **`devcheck.config.json`** — added `packaging.pluginManifests: true`, enabling the new `lint:packaging` check that validates the plugin manifests' descriptions and scoped/unscoped identity fields.
- **Framework devcheck scripts** — synced to mcp-ts-core ^0.10.9, adding `scripts/check-dependency-specifiers.ts` (hard-fails on floating `latest`/`*`/dist-tag specifiers in `package.json` and the `bun.lock` workspace map) and the fresh-scaffold git/changelog guards.
- **Project skills** — 14 `skills/<name>/SKILL.md` files synced from mcp-ts-core ^0.10.9.

## Fixed

- **`secedgar_dataframe_describe` filtered describe** — via mcp-ts-core ^0.10.9, `DuckdbProvider.describe({ tableName })` qualifies its pushed `WHERE` filters with the table alias, fixing the DuckDB binder error ("Ambiguous reference to column name 'table_name'") that previously broke every name-filtered describe. The 0.11.0 server-side workaround (unfiltered `describe()` + in-memory `.find()`) is no longer compensating for an upstream bug. ([mcp-ts-core #235](https://github.com/cyanheads/mcp-ts-core/issues/235))
