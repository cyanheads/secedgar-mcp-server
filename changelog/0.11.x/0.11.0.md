# 0.11.0 — 2026-06-11

## Added

- **`secedgar_get_filing` `offset` param** — character offset into extracted document text; default `0`. Pass `next_offset` from a truncated response to read the next page. (#59)
- **`secedgar_get_filing` `section` param** — case-insensitive substring match against detected headings (e.g. `"risk factors"`, `"item 7"`); resolves to a heading offset, takes precedence over `offset`. On a miss, the error data carries the detected outline. (#59)
- **`secedgar_get_filing` `next_offset` output field** — character offset for the next page; present only when `content_truncated` is true. (#59)
- **`secedgar_get_filing` `outline` output field** — detected headings with offsets; emitted on the first truncated page (`offset=0`, no `section`). Use a heading offset as `offset` or pass heading text as `section`. (#59)
- **`filingToExtract`** — new export from `filing-to-text.ts`; pure text extraction step (HTML → plain text, no limit). (#59)
- **`windowText`** — new export from `filing-to-text.ts`; windows pre-extracted text from `effectiveOffset` to `effectiveOffset + limit`, truncating at a word boundary. Returns `{ text, truncated, totalLength, nextOffset? }`. (#59)
- **`detectHeadings`** — new export from `filing-to-text.ts`; regex scan for `ITEM N` and all-caps headings with character offsets. Deduplicates by heading text, keeping the later (body) occurrence. (#59)
- **Extraction cache** — module-level 8-entry LRU in `filing-to-text.ts`, keyed by `accession:document`. SEC filings are immutable; cache hit skips both the document fetch and `convert()`. Exports: `getExtractCache`, `setExtractCache`, `clearExtractCache`, `extractCacheSize`. (#59)
- **`isSelectShaped`** — new export from `sql-gate-extras.ts`; returns true when SQL (after stripping string literals and comments) starts with `SELECT` or `WITH`. Used to distinguish a column-not-found prepare failure from a legitimately non-SELECT statement. (#52)

## Changed

- **`secedgar_get_filing` description** — updated to mention `offset`/`next_offset` paging and `section` targeting; `content_limit` description clarified as per-page size. (#59)
- **`secedgar_dataframe_query` `register_as` Zod schema** — added `.regex(/^df_[A-Z0-9]{5}_[A-Z0-9]{5}$/)` to enforce the documented `df_XXXXX_XXXXX` shape at validation time. (#53)
- **`CanvasBridge.query` catch block** — extended to intercept `McpError`s with `reason: 'missing_table'` and rebuild the error with agent-facing tool guidance instead of internal SDK method names; also reclassifies `reason: 'non_select_statement'` + `statementType: 'UNKNOWN'` when the SQL is SELECT-shaped into `reason: 'invalid_sql'` pointing to `secedgar_dataframe_describe`. (#52, #54)
- **`CanvasBridge` `describe()` call after `registerAs`** — replaced `instance.describe({ tableName })` with unfiltered `instance.describe()` + `.find()` by name to work around a DuckDB binder error ("Ambiguous reference to column name 'table_name'") in the framework's `describe()` WHERE clause. (#53)
- **`secedgar_company_search` `query` Zod schema** — added `.trim()` before `.min(1)` so whitespace-only inputs fail validation. (#57)
- **`secedgar_search_filings` `query` Zod schema** — added `.trim()` before `.min(1)` so whitespace-only inputs fail validation. (#57)
- **`secedgar_company_search` bare-CIK fallback** — numeric queries that don't resolve to an existing company now return `reason: 'no_match'` instead of a raw 404 with an internal SEC API URL. (#55)
- **`secedgar_search_filings` `offset` description** — clarified that date-sort and entity-targeted searches fetch a single 100-row window; offsets past the window return nothing; `sort=relevance` supports EDGAR-side pagination up to 10,000. (#56)
- **`secedgar_search_filings` zero-result notice** — switched check from `results.length === 0` to `total === 0` (genuine no-match only); echoes query, forms, and date range. Added a separate notice when `offset` exceeds the date-sort window. (#56, #58)

## Fixed

- **`secedgar_dataframe_query`**: column-not-found errors no longer return `non_select_statement` for SELECT-shaped queries. (#52)
- **`secedgar_dataframe_query` `register_as`**: always failed with "Ambiguous reference to column name 'table_name'"; `register_as` is now functional. (#53)
- **`secedgar_dataframe_query`**: `missing_table` recovery hint no longer leaks internal SDK method names (`registerTable()`, `describe()`). (#54)
- **`secedgar_company_search`**: numeric CIK queries that don't exist now surface `reason: 'no_match'` with recovery hint instead of a raw 404. (#55)
- **`secedgar_search_filings`**: whitespace-only query now rejected at validation instead of matching arbitrary companies via empty-prefix. (#57)
- **`secedgar_company_search`**: whitespace-only query now rejected at validation instead of matching arbitrary companies via empty-prefix. (#57)
- **`secedgar_search_filings`**: zero-result notice no longer fires for offset-past-window cases where `total > 0`. (#56, #58)
