# 0.11.3 — 2026-07-02

## Fixed

- **`register_as` clash surfaces structured** — `CanvasBridge.query` pre-checks a `register_as` target against tracked dataframes before the provider runs (works around [cyanheads/mcp-ts-core#254](https://github.com/cyanheads/mcp-ts-core/issues/254), where `DuckdbProvider`'s catch strips the framework clash error's structured data), and rebuilds escaping framework `register_as_clash` / `system_catalog_access` errors with the declared recovery hints; `register_as_clash` added to the `secedgar_dataframe_query` `errors[]` contract. ([#60](https://github.com/cyanheads/secedgar-mcp-server/issues/60))
- **EFTS degraded-2xx guard + typed targeting errors** — `EdgarApiService.searchFilings` throws a structured retryable `ServiceUnavailable` (`efts_degraded_response`) when EFTS returns a 2xx body without `hits.total` (degraded payload or an `{ error: … }` echo) instead of leaking a raw TypeError; an unresolved `ticker:` or non-numeric `cik:` token in `secedgar_search_filings` now fails with typed `unresolved_ticker` / `invalid_cik` instead of stripping the token and searching unscoped. ([#61](https://github.com/cyanheads/secedgar-mcp-server/issues/61))
- **Insider-transactions canvas scan floor** — with a canvas available, `secedgar_get_insider_transactions` fetches at least the 40-filing scan floor (capped at 100) regardless of the inline `limit`, plus a +1 sentinel row that detects Form 4 filings beyond the window — so small limits no longer under-scan the dataframe and `dataset.truncated` is truthful. ([#63](https://github.com/cyanheads/secedgar-mcp-server/issues/63))
- **`get_institutional_holdings` scope corrected in docs** — the tool description, input description, and README no longer advertise a reverse issuer-to-holders lookup (EDGAR has no issuer-to-13F index); `ticker_or_cik` is documented as the institutional filer only, with issuer-side questions routed to `secedgar_search_filings` with `forms=["13F-HR"]`. ([#62](https://github.com/cyanheads/secedgar-mcp-server/issues/62))
- **`.env.example` dataframe settings** — new dataframes section documents `EDGAR_DATASET_TTL_SECONDS` and `EDGAR_DATAFRAME_DROP_ENABLED`. ([#68](https://github.com/cyanheads/secedgar-mcp-server/issues/68))
