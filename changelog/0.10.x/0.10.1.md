# 0.10.1 — 2026-06-08

## Security

- **DataCanvas SQL gate now fails closed** ([#210](https://github.com/cyanheads/mcp-ts-core/issues/210)) — non-SELECT statements (e.g. bare `PRAGMA`) previously returned a generic `InternalError` (-32603) instead of the documented `non_select_statement` `ValidationError`; the gate now rejects anything it cannot prove is a single SELECT. The `pragma_*` table-function namespace (e.g. `SELECT * FROM pragma_table_info(...)`) is also denied, closing a bypass route. Hardens `secedgar_dataframe_query`.

## Changed

- **Boolean env vars now parse via `z.stringbool()`** ([#201](https://github.com/cyanheads/mcp-ts-core/issues/201)) — `EDGAR_DATAFRAME_DROP_ENABLED`, `EDGAR_MIRROR_ENABLED`, and `EDGAR_MIRROR_FALLBACK_LIVE` now accept `true/false/1/0/yes/no/on/off` (case-insensitive) and reject unrecognized values at startup instead of silently coercing them to `false`. Defaults unchanged.
- **`.mcpbignore` dev-dir patterns re-anchored to root** ([#207](https://github.com/cyanheads/mcp-ts-core/issues/207)) — `skills/`, `.agents/`, `.claude/` entries are now root-anchored so they cannot accidentally strip nested runtime paths (e.g. paths containing `skills/` deeper in the tree).
- **Missing `node-cron` peer throws an actionable error** ([#200](https://github.com/cyanheads/mcp-ts-core/issues/200)) — the mirror refresh scheduler no longer crash-loops the server with a raw module-not-found; it now throws a `configurationError` naming the required peer (`node-cron ^4.2.1`).

## Dependencies

- `@cyanheads/mcp-ts-core` ^0.9.21 → ^0.10.1
