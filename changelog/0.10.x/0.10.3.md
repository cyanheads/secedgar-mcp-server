# 0.10.3 — 2026-06-11

## Changed

- **`CanvasBridge.register()`** — passes `ttlMs` via `RegisterTableOptions.ttlMs` to the framework so DuckDB-level TTL is managed natively (mcp-ts-core#140, resolved in 0.10.4). Bridge retains per-table metadata for provenance and lazy-sweep.
- **`CanvasBridge.query()`** — passes `denySystemCatalogs: true` in `QueryOptions`, delegating system-catalog denial (`information_schema`, `pg_catalog`, `sqlite_master`, `duckdb_*`) to the framework layer (mcp-ts-core#224). Local `assertNoSystemCatalogAccess` removed from `sql-gate-extras.ts`.
- **`CanvasBridge.register()`** — replaces the local `deriveAllNullableSchema` wrapper with a direct call to `inferSchemaFromRows`; the framework now always emits `nullable: true` on inferred columns since mcp-ts-core 0.10.4 (mcp-ts-core#221), so the local force-nullable pass is no longer needed.
- **`secedgar_dataframe_query` missing-table path** — framework now surfaces DuckDB binder/catalog errors for missing tables as `NotFound` with `data.reason: 'missing_table'` (mcp-ts-core#223); tool declares `missing_table` in its errors contract, and the error envelope sent to clients carries `reason: 'missing_table'`.
- **`createApp()`** — passes `title: 'SEC EDGAR MCP Server'` and `websiteUrl: 'https://github.com/cyanheads/secedgar-mcp-server'`; surfaced in `initialize` serverInfo and `/.well-known/mcp.json` (mcp-ts-core#213).
- **`ctx.enrich.truncated()`** adopted on five cap-bounded list tools: `secedgar_search_filings`, `secedgar_get_financials`, `secedgar_fetch_frames`, `secedgar_get_insider_transactions`, `secedgar_get_institutional_holdings`. Each now fires `ctx.enrich.truncated({ shown, cap })` when the inline response was capped and exposes `truncated`, `shown`, `cap` in its enrichment contract.
