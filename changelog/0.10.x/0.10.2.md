# 0.10.2 — 2026-06-08

## Fixed

- **`secedgar_get_financials`** multi-tag frame collisions now resolve by tag priority — the lower-index tag wins (preferred total over sub-line, e.g. IFRS `Revenue` over `RevenueFromContractsWithCustomers`), with ties within one tag breaking on latest `filed`; fixes silent wrong-magnitude series (Spotify IFRS `Revenue` was ~26× understated). Tag-array order in `concept-map.ts` is now semantic ([#44](https://github.com/cyanheads/secedgar-mcp-server/issues/44)).
- **`secedgar_get_financials`** instant concepts (balance-sheet fields, `shares_outstanding`, raw instant XBRL tags) now return data on the first call when `period_type` is unset, via a ground-truth post-fetch fallback that detects an all-instant series and serves it in full; replaces the prior `balance_sheet`-group proxy that missed raw instant tags ([#48](https://github.com/cyanheads/secedgar-mcp-server/issues/48)).
- **`secedgar_fetch_frames`** returns a structured `no_data` error instead of "unknown concept" for well-formed but empty or deprecated raw XBRL tags, so `unqueried_tags` suggestions from `secedgar_search_concepts` compose mechanically ([#45](https://github.com/cyanheads/secedgar-mcp-server/issues/45)).
- **`secedgar_dataframe_query`** surfaces a structured `missing_table` error (pointing to `secedgar_dataframe_describe`) for an unregistered or TTL-expired `df_<id>` handle, and a generic `invalid_sql` for other raw DuckDB execution errors; DuckDB's internal "Did you mean …?" hints are stripped to prevent catalog name leakage ([#47](https://github.com/cyanheads/secedgar-mcp-server/issues/47)).
- **`secedgar_company_search`** ETF and mutual-fund tickers (VOO, SCHD, JEPI) now resolve on mirror-enabled deployments via a live MF file merge gated on `EDGAR_MIRROR_FALLBACK_LIVE`, covering mirrors that predate MF ingestion ([#43](https://github.com/cyanheads/secedgar-mcp-server/issues/43)).

## Changed

- **`secedgar_get_insider_transactions`** `shares_traded` is now an unsigned magnitude paired with a new `direction` field (`acquire` or `dispose`) on both the inline response and the registered dataframe; compute net activity with `SUM(CASE WHEN direction='dispose' THEN -shares_traded ELSE shares_traded END)` ([#46](https://github.com/cyanheads/secedgar-mcp-server/issues/46)).

## Added

- **`secedgar_fetch_frames`** appends a dispersion caveat to the response when value dispersion (`max_to_p95_ratio`) crosses a unit-aware threshold (~50× for per-share values, ~200× otherwise), signaling likely unit-mix in the frames data ([#49](https://github.com/cyanheads/secedgar-mcp-server/issues/49)).
