# 0.5.0 — 2026-05-17

In-conversation SQL analytics. Three data-returning tools now spill their full upstream response onto a shared DuckDB canvas, addressable by `df_<id>` and queryable via standard DuckDB SQL across the new `secedgar_dataframe_*` tools. Closes [#17](https://github.com/cyanheads/secedgar-mcp-server/issues/17).

## Added

- **`secedgar_fetch_frames`** — replaces `secedgar_compare_metric`. Same friendly-concept × calendar-period × unit input, but the inline response now carries `value_distribution` (median, p95, max, max_to_p95_ratio), `period_end_range` (min/max across the frame, surfaces fiscal-year mixing), and `unqueried_tags` (other XBRL tags in the same friendly-name mapping the call did NOT query, so the LLM can fan out and UNION/COALESCE).
- **`secedgar_dataframe_describe`** — list canvas dataframes with `source_tool`, `query_params`, `created_at`, `expires_at`, `row_count`, `column_schema`, and `truncated`.
- **`secedgar_dataframe_query`** — single-statement SELECT against the shared canvas. Framework's three-layer SQL gate (single-statement → SELECT only → plan-walk allowlist) plus a bridge-layer deny on `information_schema`, `pg_catalog`, `sqlite_master`, and `duckdb_*` system catalogs so callers can't enumerate `df_<id>` handles they don't already hold. Optional `register_as` chains the result as a new dataframe with a fresh per-table TTL.
- **`secedgar_dataframe_drop`** — idempotent drop. Opt-in via `EDGAR_DATAFRAME_DROP_ENABLED=true`; off by default since TTL already handles cleanup. The only destructive tool on this server.
- **`dataset` field on `secedgar_search_filings`, `secedgar_get_financials`, `secedgar_fetch_frames`** — each call returns a `df_XXXXX_XXXXX` handle, row count, and ISO 8601 `expires_at`. `secedgar_search_filings` also reports `truncated: true` when EFTS has more text matches than the response window. Absent when no canvas is wired (e.g. Workers, `CANVAS_PROVIDER_TYPE=none`), when materialization failed, or when the inline response already fits.
- **`ticker` field on `secedgar_search_filings` results** — primary ticker parsed from the EFTS `display_names[0]` parenthetical, with hyphen-aware regex for multi-class issuers (BRK-A / BRK-B). Absent for private filers, foreign filers without a US listing, and filings whose display name omits a ticker parenthetical.
- **`CanvasBridge`** — `src/services/canvas-bridge/canvas-bridge.ts`. One shared canvas per tenant via `ctx.state['canvas-id']`. Tools call `bridge.registerDataframe({ rows, sourceTool, queryParams })` to mint `df_XXXXX_XXXXX` (~3.7×10^15 keyspace) and persist provenance + per-table TTL (`EDGAR_DATASET_TTL_SECONDS`, default 86400) under `df-meta/<table>`. Lazy TTL sweep on every public op. All-nullable column schemas via `deriveAllNullableSchema` — sparse SEC columns (`loc`, `ticker`, `start`) must not trip DuckDB's NOT NULL appender rollback.
- **`sql-gate-extras`** — `src/services/canvas-bridge/sql-gate-extras.ts`. `assertNoSystemCatalogAccess` strips string literals first, then rejects `information_schema`, `pg_catalog`, `sqlite_master`, `duckdb_*` references. Layered on top of the framework's plan-walk gate.
- **`EDGAR_DATASET_TTL_SECONDS`** (default `86400`) — per-table TTL for canvas-registered dataframes. Bridge-side bookkeeping in `ctx.state`; backstop for [cyanheads/mcp-ts-core#140](https://github.com/cyanheads/mcp-ts-core/issues/140) until the framework exposes `RegisterTableOptions.ttlMs`.
- **`EDGAR_DATAFRAME_DROP_ENABLED`** (default `false`) — gates registration of `secedgar_dataframe_drop`.
- **`CANVAS_PROVIDER_TYPE`** (default `duckdb`) — `src/index.ts` opts the canvas on by default (DuckDB ships as a direct dep). Set to `none` on Cloudflare Workers, where DuckDB has no V8-isolate build.
- **`@duckdb/node-api ^1.5.2-r.1`** direct dependency — DuckDB is now in the install tree, not an optional peer.

## Changed

- **`EdgarApiService.getTickerCache` is now singleflight.** Concurrent first-time callers (e.g. `secedgar_fetch_frames` enriching ~5k reporters in parallel) share one in-flight `loadTickerCache()` instead of each queuing their own SEC fetch through the 10 req/s throttle. Cache key timestamp now uses post-fetch `Date.now()` so the TTL window measures cached lifetime, not wall-clock from before the request.
- **`secedgar_search_filings` `cleanCompanyName` regex** allows hyphens in the trailing parenthetical to keep multi-class ticker symbols (BRK-A, BRK-B) intact during stripping. Format string now renders `(TICKER, CIK …)` when a ticker is parsed, falling back to `(CIK …)` otherwise.
- **`FrameEntry.start`** field added to the EDGAR types — present on duration frames, absent on instant frames. Surfaced in the canvas row as `period_start`.
- **`createApp({ setup })`** signature in `src/index.ts` now takes the framework `core` argument and passes `core.canvas` to `initCanvasBridge`.
- **`instructions` orientation string** updated for the new tool surface: `secedgar_compare_metric` reference replaced with `secedgar_fetch_frames`, plus a sentence on the `df_<id>` materialization flow and `secedgar_dataframe_query` / `secedgar_dataframe_describe`.

## Removed

- **`secedgar_compare_metric`** — renamed to `secedgar_fetch_frames`. Wire signature changed: `data[]` rows still carry `rank`, `company_name`, `cik`, `ticker`, `value`, `period_end`, `accession_number`, but `location` is now optional (absent when SEC has no location for the filer), and the inline response adds `value_distribution`, `period_end_range`, `unqueried_tags`, and the optional `dataset` handle. Callers should migrate the tool name and add handling for the new fields.
