# 0.4.5 — 2026-05-16

## Added

- **`createApp({ instructions })`** — server now ships a one-paragraph orientation string on every MCP `initialize` response, summarizing the `secedgar_*` tool surface (entity resolution via `secedgar_company_search`, filing fetch via `secedgar_get_filing`, full-text search with inline `ticker:` / `cik:` targeting via `secedgar_search_filings`, friendly-name XBRL via `secedgar_get_financials` / `secedgar_compare_metric` / `secedgar_search_concepts`). Spec-compliant MCP clients forward this to the model as session-level system context.

## Changed

- **Dependencies** — `@cyanheads/mcp-ts-core` `^0.8.19 → ^0.9.1`, `@biomejs/biome` `^2.4.14 → ^2.4.15`, `@types/node` `^25.6.2 → ^25.8.0`, `vitest` `^4.1.5 → ^4.1.6`. Framework `0.9.x` adds the `instructions` field, the cross-vendor `schema-*-portability` lint family, RFC 8414 §3 well-known mount, SSRF hardening (DNS validation + IPv4-mapped IPv6 / link-local / multicast blocklist), tenant-id boundary validation, and bounded rate-limit metric cardinality; full notes in `node_modules/@cyanheads/mcp-ts-core/changelog/0.9.x/`.
- **Definition linting now build-time only.** `createApp()` no longer runs `validateDefinitions` at startup — lint runs via `bun run lint:mcp` and `bun run devcheck` instead. Faster cold starts; new framework lint rules are additive and won't break deployed servers.
- **`scripts/devcheck.ts` `bun outdated` parser fixed.** Reads index `[1]` of the markdown-style row (was `[0]`, always empty) and strips the `(dev|peer|prod|optional)` workspace marker so bare package names match the allowlist.
- **Changelog summary cap raised `250 → 350` chars** in `scripts/build-changelog.ts` and `changelog/template.md`. Matches the upstream framework cap.
- **Skills sync** — refreshed `add-tool`, `api-auth`, `api-config`, `api-errors`, `api-linter`, `api-workers`, `design-mcp-server`, `field-test`, `polish-docs-meta` (including `references/package-meta.md` and `references/readme.md`), `security-pass`, `tool-defs-analysis`. `CLAUDE.md` / `AGENTS.md` Checklist tightened: explicit list of non-serializable Zod types, `schema-format-portability` callout for `z.url()` / `z.cuid()` / `z.base64()` / `z.jwt()`, and the `z.union([z.literal(''), z.string().regex(...).describe(...)])` recipe for regex-constrained optional fields under form-based clients.
