# 0.4.2 — 2026-05-04

Framework upgrade to `@cyanheads/mcp-ts-core` 0.8.13 and an internal coherence pass on `secedgar_get_filing`'s error path so every declared contract reason throws via `ctx.fail` from handler-visible code. No behavioural change on the wire — same JSON-RPC error codes, same `data.reason`, same recovery hints.

## Changed

- **`secedgar_get_filing` archive helper threads `ctx`** — `resolveFilingArchive` now accepts a typed `HandlerContext<ReasonOf<typeof FILING_ERRORS>>` and throws via `ctx.fail('document_not_found' | 'no_documents' | 'filing_not_found', …)` instead of `new McpError(JsonRpcErrorCode.NotFound, …, { reason, recovery })`. The `errors[]` contract is hoisted to a module-level `FILING_ERRORS` const so the helper's `ctx` parameter can be typed against the reason union via the framework's `ReasonOf` utility. Brings the tool in line with the other five contract-bearing tools and lets the framework's `error-contract-prefer-fail` lint actually enforce conformance — previously the lint couldn't see throws from helpers.
- **`@cyanheads/mcp-ts-core` `^0.8.11` → `^0.8.13`** (two patches). Both releases are Canvas-focused (SQL gate hardening for `read_json*`/`read_parquet*` file-disclosure bypass, typed `appendValue` for TIMESTAMP/DATE/BLOB, new `registerView` / `importFrom` primitives, refreshed DuckDB v1.5.x plan-operator allowlist, spatial-extension pre-staging) — none apply here since this server doesn't use Canvas. Universal carry-over: vitest `silent: 'passed-only'` and node-cron sourcemap-warning suppression flow in via the shared `coreConfig` import, no project changes needed.
- **Refreshed `api-canvas` skill** from the package (1.0 → 1.1). Other 27 skills already at package version. All 10 framework `scripts/*.ts` unchanged.
