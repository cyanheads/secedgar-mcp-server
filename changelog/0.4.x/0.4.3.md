# 0.4.3 — 2026-05-04

Two long-standing usability bugs on the document-fetching path are fixed: `secedgar_search_filings` now sorts by filing date by default (closes [#5](https://github.com/cyanheads/secedgar-mcp-server/issues/5)) and `secedgar_get_filing`'s `documents` field is now categorized rather than a flat list dominated by XBRL viewer artifacts (closes [#3](https://github.com/cyanheads/secedgar-mcp-server/issues/3)). Both ship with new optional inputs and richer schema descriptions.

## Added

- **`secedgar_search_filings` — `sort` input** with values `filing_date_desc` (default), `filing_date_asc`, `relevance`. EFTS exposes no native sort knob and ranks purely by relevance, so date sorts re-order the EFTS top-100 sample client-side. For broad queries with more than 100 EFTS matches and no entity targeting, absolute newest filings can sit outside that window — schema documents the bound and recommends `ticker:` / `cik:` targeting or query tightening when absolute recency matters. Pure relevance mode preserves the prior pass-through behaviour for callers that want it.
- **`secedgar_get_filing` — `include_xbrl` input** (default `false`) gates XBRL viewer fragments and machine-readable taxonomy files (`R*.htm`, `*_cal/_def/_lab/_pre.xml`, `*_htm.xml`, `*.xsd`, `MetaLinks.json`, `FilingSummary.xml`, `Show.js`, `report.css`, `*-xbrl.zip`, `Financial_Report.xlsx`, `EX-101.*`) under a separate `documents.xbrl` bucket. A typical 10-K archive contains ~100 such entries; suppressing them by default removes most of the noise from the response.
- **`EdgarApiService.tryGetFilingHeaders(cik, accession)`** + new `services/edgar/filing-headers.ts` parser. Fetches `<accession>-index-headers.html` (the HTML re-encoding of the original SGML submission header) and parses each `<DOCUMENT>` block into a `filename → { type, sequence, description }` map. The header file exposes canonical SEC TYPE values like `EX-21.1`, `EX-31.1`, `GRAPHIC` that the directory-listing JSON does not — `index.json` only returns icon hints (`text.gif`). Best-effort: returns `null` when the header file is absent.

## Changed

- **`secedgar_get_filing` — `documents` is now an object, not an array.** Schema groups entries into `primary` / `exhibits` / `auxiliary` / `xbrl` (the last only present when `include_xbrl=true`). Each entry carries SEC's canonical `type` from the submission header and an optional `description` (e.g. `"Subsidiaries of the Registrant"`). When the header file is unavailable, `type` falls back to a name-pattern label (`XBRL-LINKBASE`, `XBRL-INSTANCE`, `XBRL-VIEWER`, etc.) and exhibits whose filename does not match a known pattern fall through to `auxiliary`. Output schema change — clients that previously iterated `documents[]` need to read `documents.primary`, `documents.exhibits`, `documents.auxiliary`.
- **`secedgar_get_filing` — submissions metadata + filing-headers fetch parallelized** via `Promise.all`. Adds one HTTP roundtrip per call but runs alongside the existing submissions fetch, so wall-clock cost is roughly nil.
- **`secedgar_search_filings` — `total_is_exact` is now correctly `false` under entity targeting** when EFTS returned more matches than the sample window covered. Previously the post-filter unconditionally set `total_is_exact = true` even when additional entity hits could exist beyond the 100-hit sample.
- **`secedgar_compare_metric` — `unit` accepts `USD/shares` in addition to `USD-per-shares`.** SEC's frames API path requires the dashed form, but the slash form is more natural for callers and matches the unit string returned by `secedgar_search_concepts`. Either form is normalized to the dashed form before the lookup.
- **Schema descriptions polished across all six tools.** Optional output fields now name the conditions under which they're absent (e.g. "Absent for filings older than the last ~1,000 the company has filed", "Absent when SEC has no location for this filer") rather than vague "Optional." labels. Cross-tool references added where workflows chain (`secedgar_company_search` → `secedgar_get_filing`, `secedgar_search_concepts` → `secedgar_get_financials` / `secedgar_compare_metric`).
- **`secedgar_company_analysis` prompt** points step 4 at `secedgar_search_filings` and `secedgar_company_search` with `form_types: ["8-K"]` for material-events discovery, rather than referring to a generic "search for 8-K filings" without a tool name.

## Fixed

- **[#3](https://github.com/cyanheads/secedgar-mcp-server/issues/3) — `secedgar_get_filing` documents array bloated with XBRL viewer artifacts.** A typical 10-K archive has ~110 entries, ~95% of which are XBRL rendering fragments. Categorization + the `include_xbrl=false` default removes the bloat from the default response.
- **[#5](https://github.com/cyanheads/secedgar-mcp-server/issues/5) — `secedgar_search_filings` result sort order unpredictable.** Default sort is now `filing_date_desc` (most recent first); `filing_date_asc` and `relevance` available as overrides.
