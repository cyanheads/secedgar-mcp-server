# 0.4.4 — 2026-05-08

## Changed

- **`secedgar_get_filing` — handler owns all error throws.** `resolveFilingArchive` returns a discriminated outcome union (`document_not_found | no_documents | filing_not_found | ok`); the handler maps each to `ctx.fail`. Pure refactor — wire format and contract unchanged.
- **Schema descriptions tightened across all six tools and both resources.** Date fields spell out `YYYY-MM-DD`; optional output fields name the conditions under which they're absent (e.g. `report_date` "Absent for filings without a reporting period (proxy statements, ownership reports)"); `accession_number` notes its dash format; cross-tool references use full `secedgar_*` names.
- **Engines** — `node` minimum bumped `>=22.0.0 → >=24.0.0`; Dockerfile base image pinned `oven/bun:1 → oven/bun:1.3` for build and runtime stages.
- **Dependencies** — `@cyanheads/mcp-ts-core ^0.8.13 → ^0.8.19`, `@types/node ^25.6.0 → ^25.6.2`.
- **Changelog tooling — `security: true` frontmatter flag** with a `🛡️ Security` badge in the rollup, rendered alongside the existing `⚠️ Breaking` badge (Breaking before Security when both set). `changelog/template.md` rewritten as a format reference plus authoring guide for new release files.
- **Skills sync** — refreshed `skills/`: `add-tool`, `api-canvas`, `api-context`, `api-errors`, `api-utils`, `api-workers`, `design-mcp-server`, `maintenance`, `report-issue-framework`, `report-issue-local`, `setup`. Added `api-telemetry` (OTel catalog) and `tool-defs-analysis` (read-only definition-language audit). `AGENTS.md` / `CLAUDE.md` skill table updated to match.

## Fixed

- **`secedgar_search_filings` — `form_distribution` now matches `total` under entity targeting.** When `ticker:` or `cik:` in the query filtered the EFTS sample client-side, `form_distribution` previously reflected EFTS's pre-filter aggregation buckets and could disagree with the post-filter `total`. The distribution now rebuilds from the filtered hits whenever entity targeting is active.
