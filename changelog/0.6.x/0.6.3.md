# 0.6.3 — 2026-05-30

## Added

- **`enrichment` block** on `secedgar_search_filings`, `secedgar_search_concepts`, `secedgar_company_search`, and `secedgar_dataframe_query` — populates via `ctx.enrich` so agent-facing context reaches both `structuredContent` JSON and the `content[]` markdown trailer automatically.
  - `secedgar_search_filings`: echoes `effectiveQuery` (the query as EDGAR executed it) and surfaces a `notice` with empty-result guidance including entity targeting syntax hints.
  - `secedgar_search_concepts`: surfaces a `notice` with empty-result guidance including applied filters and a full-catalog fallback suggestion.
  - `secedgar_company_search`: surfaces a `notice` when `include_filings=true` but no filings matched the `form_types` filter.
  - `secedgar_dataframe_query`: surfaces a `notice` for 0-row results (prompts `secedgar_dataframe_describe` verification) and row-cap hits (suggests `register_as` or raising `row_limit`).
- `dataframe-query.tool.test.ts` and `search-concepts.tool.test.ts` — enrichment and edge-case coverage for the two newly-enriched tools.

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.13 → ^0.9.16` — picks up `ctx.enrich` API, enrichment lint rules, and stale-skill detection in the skills-sync check.
- **`secedgar_search_concepts` `format()`** — empty-result path now returns `Found 0 concepts.` (plain count); the actionable guidance moves to the `enrichment.notice` channel instead of the markdown body.
- **Skills** updated to `^0.9.16` framework: `add-tool`, `add-app-tool`, `add-service`, `api-context` (ctx.enrich API), `api-linter` (enrichment rules), `design-mcp-server`, `git-wrapup`, `maintenance`, `polish-docs-meta`, `setup`; `devcheck` skill removed (merged into framework devcheck).
- **`format` / `format:unsafe`** split into two scripts in `package.json` — `format` runs safe Biome fixes; `format:unsafe` runs unsafe rewrites.
