# 0.6.2 — 2026-05-28

## Added

- **`.claude-plugin/`** and **`.codex-plugin/`** — plugin metadata files for Claude Code and Codex plugin discovery.
- **`skills/code-simplifier/`** — post-session code review and cleanup skill synced from framework 0.9.11.
- **`MCP_HTTP_MAX_BODY_BYTES`** — inbound request-body cap on the HTTP MCP endpoint (default 1 MiB). Oversized requests are rejected with `413` before the request body is parsed. Set to `0` to disable. Documented in `.env.example`. (via mcp-ts-core 0.9.13, [#157](https://github.com/cyanheads/mcp-ts-core/issues/157))

## Changed

- **`@cyanheads/mcp-ts-core` ^0.9.9 → ^0.9.13** — four framework releases:
  - 0.9.10: HTTP session-init gate — stateful HTTP mode rejects non-`initialize` requests without `Mcp-Session-Id` with HTTP 400 ([#154](https://github.com/cyanheads/mcp-ts-core/issues/154)); expected client errors (401, 403, 400, 404) now log at `warning` level without stack traces ([#158](https://github.com/cyanheads/mcp-ts-core/issues/158))
  - 0.9.11: `code-simplifier` skill added; `mcp-ts-core init --help` flag added
  - 0.9.12: `GET /mcp` status JSON now includes `package.json` `keywords` ([#165](https://github.com/cyanheads/mcp-ts-core/issues/165))
  - 0.9.13: `MCP_HTTP_MAX_BODY_BYTES` 413 body cap; `landing.requireAuth` defaults to `true` when auth mode is `jwt` or `oauth` ([#157](https://github.com/cyanheads/mcp-ts-core/issues/157), [#156](https://github.com/cyanheads/mcp-ts-core/issues/156))
- **`@biomejs/biome` ^2.4.15 → ^2.4.16**
- **`manifest.json`** — `repository`, `homepage`, and `license` fields added.
- **`skills/`** — `api-canvas`, `api-config`, `design-mcp-server`, `git-wrapup`, `polish-docs-meta`, `release-and-publish`, `report-issue-framework` updated from framework sync.
- **`CLAUDE.md` / `AGENTS.md`** — plugin file checklist items and skill table entries added.
- **`changelog/template.md`** — `agent-notes` frontmatter field documented.

## Security

- **HTTP request-body cap** — `MCP_HTTP_MAX_BODY_BYTES` (default 1 MiB) rejects oversized bodies with `413` before parsing, guarding against request-body amplification on the HTTP transport.

## Removed

- **`skills/migrate-mcp-ts-template/SKILL.md`** — migration-era skill removed (framework 0.9.12).
