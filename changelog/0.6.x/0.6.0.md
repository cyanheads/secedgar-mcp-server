# 0.6.0 — 2026-05-25

## Added

- **`concept-map.ts` expanded from 16 → 33 entries** — 17 new friendly names across income statement (`cogs`, `rd_expense`, `sga_expense`, `interest_expense`, `tax_expense`, `stock_based_compensation`), balance sheet (`current_assets`, `current_liabilities`, `inventory`, `accounts_receivable`, `accounts_payable`, `goodwill`, `intangible_assets`), and cash flow (`dividends_paid`, `share_repurchases`, `investing_cash_flow`, `financing_cash_flow`). ([#12](https://github.com/cyanheads/secedgar-mcp-server/issues/12))
- **`get_financials` probes `companyfacts` on `no_concept_data`** — when no XBRL data is found, surfaces actual namespaces and matching tags from the filer's `companyfacts` payload so callers have concrete fallback options. `available_namespaces` added to error data. Adds `EdgarApiService.tryGetCompanyFacts`. ([#13](https://github.com/cyanheads/secedgar-mcp-server/issues/13))
- **`get_filing` adds `requested_document` field** — present only when the document param targets an exhibit that differs from `primary_document`. ([#4](https://github.com/cyanheads/secedgar-mcp-server/issues/4))

## Changed

- **`get_financials` fiscal context format** — changed from the bracket notation `[FY2023 FY]` to a parenthetical inline with the form: `10-K (FY2023 FY)`. ([#11](https://github.com/cyanheads/secedgar-mcp-server/issues/11))
- **`company_search` `fiscal_year_end`** — formatted as `MM-DD` (e.g. `09-26`) instead of the raw `MMDD` string from the SEC API. ([#14](https://github.com/cyanheads/secedgar-mcp-server/issues/14))
- **`search_filings` `forms` param description** — notes that amendment types (e.g. `10-K/A`) are included when filtering by base form. ([#14](https://github.com/cyanheads/secedgar-mcp-server/issues/14))

## Fixed

- **`get_filing` `primary_document`** — now always reflects the filing's actual primary document regardless of the `document` param. Previously, fetching an exhibit overwrote `primary_document` with the exhibit filename. ([#4](https://github.com/cyanheads/secedgar-mcp-server/issues/4))
- **`search_filings` `form_distribution`** — was recomputed from post-filter hits only when entity targeting was active. Now also recomputed when a `forms` filter is applied, keeping counts consistent with `total`. ([#6](https://github.com/cyanheads/secedgar-mcp-server/issues/6))
