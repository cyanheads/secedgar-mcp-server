# 0.9.0 — 2026-06-08

## Added

- **`secedgar_get_insider_transactions` canvas output** — when a canvas is present, the tool now materializes the full parsed Form 4 result set as a `df_<id>` dataframe (new `dataset` output field); the inline response remains a preview capped at `limit`. The scan covers up to `INSIDER_CANVAS_FILING_SCAN` (default 40) recent filings and flags `dataset.truncated` when the form index has more. Canvas behavior is transparent when unavailable — no extra fetches occur. ([#39](https://github.com/cyanheads/secedgar-mcp-server/issues/39))
- **`secedgar_get_institutional_holdings` canvas output** — same pattern: full info-table result (consolidated or raw per `consolidate`) registered as `df_<id>`; inline response is a preview capped at `limit`. ([#39](https://github.com/cyanheads/secedgar-mcp-server/issues/39))
- **`secedgar_dataframe_describe` now includes ownership tools** — `get_insider_transactions` and `get_institutional_holdings` appear in the tool source list alongside the three existing canvas tools.
- **`secedgar_fetch_frames` `related_tags` field** — returns alternate-definition XBRL tags that a meaningful share of filers report as their primary line for the base concept (e.g. `us-gaap/CashAndCashEquivalentsAtCarryingValue` → includes `us-gaap/CashCashEquivalentsAndShortTermInvestments`; `us-gaap/StockholdersEquity` → includes noncontrolling-interest-inclusive totals; operating-only vs. combined cash-flow variants). A whole-universe screen on the base tag alone is no longer silently under-inclusive. ([#36](https://github.com/cyanheads/secedgar-mcp-server/issues/36))
- **Alternate-definition tags in `secedgar_search_concepts` and `secedgar://concepts` resource** — each concept entry now surfaces `alternateTags` listing related definitions, and alternate tags are reverse-searchable (a lookup on an alternate tag returns its canonical concept). ([#36](https://github.com/cyanheads/secedgar-mcp-server/issues/36))
- **Alternate tags excluded from `get_financials` fallback chain** — kept out intentionally to avoid conflating semantically distinct totals; the `related_tags` in `fetch_frames` provides the explicit signal for cross-tag aggregation.

## Changed

- **`@types/node`** `^25.9.1` → `^25.9.2` (dev-dep patch).
