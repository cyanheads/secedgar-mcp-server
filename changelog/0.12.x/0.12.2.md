# 0.12.2 — 2026-07-09

## Added

- **`secedgar_get_institutional_holdings` name resolution** — `EdgarApiService.resolveEntityByName` (EFTS `search-index?keysTyped=`) resolves an institutional name when the ticker cache misses, covering managers, trusts, and individuals absent from `company_tickers.json` (#73).
- **`ambiguous_entity` error** — `secedgar_get_institutional_holdings` throws `ValidationError` (-32007) with up to 10 CIK/name candidates when a name matches multiple EDGAR entities (from the ticker cache or EFTS), instead of silently taking the first match (#73).
- **Operating-company routing** — a CIK with no 13F-HR but a 10-K, 10-Q, or 8-K (or amendment) in its recent filings is classified as an operating company; the `no_filings_found` error routes to `secedgar_get_financials`/`secedgar_search_filings` via `data.suggestions` (#86).
- **`secedgar_company_analysis` ownership routing** — `focus_areas` matching insider/management-transaction/ownership adds an "Insider Activity" step (`secedgar_get_insider_transactions`); matching institutional/13F/ownership adds "Institutional Ownership" (`secedgar_get_institutional_holdings`); "ownership" triggers both (#75).

## Changed

- **`secedgar_get_institutional_holdings`** fetches submissions directly via `EdgarApiService.getSubmissions` rather than `getRecentFilingsByForm`, so one response backs quarter matching, 404 recovery, and operating-company classification.
- Tool description and `ticker_or_cik` field describe CIK-first resolution and EDGAR entity-name search, replacing the prior "CIK or full legal name" framing.

## Fixed

- **`secedgar_get_institutional_holdings`** no longer leaks the raw `data.sec.gov` submissions URL when a ticker-cache-miss (bare) CIK has no submissions feed — the 404 converts to `company_not_found` with a recovery hint; a 404 on a cache-hit or name-resolved CIK still propagates unchanged (#76).
