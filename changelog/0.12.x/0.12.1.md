# 0.12.1 — 2026-07-06

## Added

- **Supply-chain install guard** — `bunfig.toml` gains `minimumReleaseAge = 259200` (a 3-day hold on freshly published versions; existing `bun.lock` pins are unaffected) and an `[install.security]` block registering `@socketsecurity/bun-security-scanner`, which scans every package for malware, typosquats, and CVEs before install. `@cyanheads/mcp-ts-core` is excluded from the hold via `minimumReleaseAgeExcludes` so framework releases adopt same-day.
- **`.github/SECURITY.md`** — disclosure policy: report privately via the repository Security tab or casey@caseyjhand.com; fixes ship against the latest release.
- **`.gitattributes`** — LF normalization, `binary` markers for image/`.mcpb`/`.sqlite` assets, and `linguist-generated` on `bun.lock`, `CHANGELOG.md`, and `docs/tree.md`.

## Changed

- **Dockerfile** pins `oven/bun:1.3.14` across the build and production stages and adds BuildKit cache mounts for Bun's install cache. The build stage keeps install lifecycle scripts enabled so `better-sqlite3`'s native binding compiles and is copied into the production image.
- **`devcheck` Outdated gate** skips versions held back by `minimumReleaseAge` (bun reports the update as equal to Current with a `*` marker); a genuine update still fails the gate.
- **`package.json` packaging** — `author` set to `Casey Hand <casey@caseyjhand.com>`; `packageManager` bun@1.3.11 → bun@1.3.14; the unused `main`/`types` fields dropped in favor of `exports`; `@socketsecurity/bun-security-scanner` and `better-sqlite3` added to `trustedDependencies`.
- **Changelog `security` flag** — `changelog/template.md` and `build-changelog` scope the flag to source-code security fixes only; dependency or transitive CVE bumps stay under `## Dependencies`.
- **Project skills + protocol docs** — `skills/` resynced to the mcp-ts-core 0.10.14 set; the `CLAUDE.md`/`AGENTS.md` framework reference bumped to `^0.10.14`.
