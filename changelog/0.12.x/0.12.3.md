# 0.12.3 — 2026-07-09

## Added

- **`secedgar_company_search` archive-page paging** — `filed_after`/`filed_before` (YYYY-MM-DD) route the scan into the submissions API's older `filings.files[]` pages via a new `EdgarApiService.fetchArchivePage`, reaching filings beyond the ~1000-entry `recent` window (e.g. a company's 2005 10-K). A form filter that under-fills `filing_limit` from `recent` alone also triggers the walk — newest page first, capped at 10 pages (`ARCHIVE_PAGE_SCAN_CAP`). ([#78](https://github.com/cyanheads/secedgar-mcp-server/issues/78))
- **`secedgar_company_search` scan-depth and dataset disclosure** — a new `history_scanned_through` output field reports the oldest filing date reached. When the scan crosses into archive pages and the full filtered history exceeds `filing_limit`, it registers as a `df_<id>` canvas dataframe (`dataset.name`/`row_count`/`expires_at`/`truncated`), mirroring the ownership tools' preview-plus-full-set pattern. ([#78](https://github.com/cyanheads/secedgar-mcp-server/issues/78))
- **`secedgar_search_filings` browse mode** — `query` is now optional; omit it (or pass `""`) to browse by `forms` and/or entity targeting (`ticker:`/`cik:`), optionally narrowed by date. A new `missing_criteria` error (`ValidationError`, -32007) fires when neither query text nor a `forms` filter is present — a bare date range still isn't a valid search on its own. On the no-query path, EFTS has no relevance signal, so every `sort` mode returns the same natural date-descending order. ([#79](https://github.com/cyanheads/secedgar-mcp-server/issues/79))
- **`secedgar_dataframe_query` declares `non_select_statement`** — the tool's `errors[]` contract now includes the reason (`ValidationError`, -32007) with a recovery hint pointing at `secedgar_dataframe_describe`. ([#74](https://github.com/cyanheads/secedgar-mcp-server/issues/74))

## Changed

- **`EdgarApiService.getSubmissions`** caches per CIK (TTL = `EDGAR_TICKER_CACHE_TTL`); the new `fetchArchivePage` caches fetched archive pages per page name on the same TTL, and the exported `selectArchivePages` selects and orders the pages covering a date range, or the full manifest newest-first. ([#78](https://github.com/cyanheads/secedgar-mcp-server/issues/78))
- **`secedgar_search_filings` zero-hit and deep-paging notices** name whichever criteria are actually active (query, entity CIK, forms, dates) instead of assuming a query string exists, and stop suggesting `sort=relevance` as a pagination escape when entity targeting already forces the single-window fetch. ([#79](https://github.com/cyanheads/secedgar-mcp-server/issues/79))

## Fixed

- **`CanvasBridge.query()` non-SELECT rejections carry a recovery hint** — a genuine non-SELECT (e.g. `DROP`, `INSERT`) previously propagated the framework's bare `non_select_statement` error with no `recovery.hint`; it's now rewrapped with the same guidance surfaced through the tool's declared contract. ([#74](https://github.com/cyanheads/secedgar-mcp-server/issues/74))
- **`secedgar://filing-types` lists the current `SCHEDULE 13D`/`SCHEDULE 13G` form names** — the legacy `SC 13D`/`SC 13G` names stopped appearing on new filings after the December 2024 structured-filing transition; the resource now lists the current names and notes the transition. ([#83](https://github.com/cyanheads/secedgar-mcp-server/issues/83))
