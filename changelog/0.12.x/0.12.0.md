# 0.12.0 — 2026-07-02

## Changed

- **⚠️ Breaking: `get_financials` dataframe columns renamed** — the registered dataframe's `fiscal_year`/`fiscal_period` columns are now `source_filing_fy`/`source_filing_fp`. The values identify the source filing, not the data period — every comparative period restated in the same filing carries that filing's fy/fp, so SQL that ordered, grouped, or windowed by `fiscal_year` silently produced wrong results. Migration: update SQL to the new names; order, group, and window by `period_end`. The inline `data[]` field names (`fiscal_year`/`fiscal_period`) are unchanged; their `.describe()` text and the `dataset` description now state the source-filing semantics and point at `period_end` as the time key. ([#72](https://github.com/cyanheads/secedgar-mcp-server/issues/72))

## Fixed

- **Mixed-case Item/Part heading detection** — `detectHeadings` adds a case-insensitive line-anchored regex for "Item 1A. Risk Factors" / "Part II" forms (non-breaking-space separators included) alongside the all-caps regex, so `section` targeting works on modern styled 10-K/10-Qs; bare "Item N." TOC cells and long unwrapped paragraphs starting with "Item" are excluded. Dedup is now case-insensitive (later occurrence wins) and results are offset-sorted. ([#71](https://github.com/cyanheads/secedgar-mcp-server/issues/71))
- **Section-miss outline reaches the client** — `secedgar_get_filing`'s `section_not_found` error message now embeds the detected outline (the same `[offset] HEADING` block the success path renders); the recovery hint points at the outline in the message, or at offset paging when no headings were detected. Clients only reliably see message + recovery hint, so the outline previously attached to error data never surfaced. ([#70](https://github.com/cyanheads/secedgar-mcp-server/issues/70))
