# 0.12.4 — 2026-07-09

## Added

- **`secedgar_search_filings` pre-2001 archive coverage** — date ranges entirely before 2001-01-01 (the EFTS full-text floor) now route to two archive-backed arms instead of silently missing data: an entity-scoped range (`ticker:`/`cik:`) reads the filer's full submissions history via `serveSubmissionsArchive` (reusing `getSubmissions`/`selectArchivePages`/`fetchArchivePage` from [#78](https://github.com/cyanheads/secedgar-mcp-server/issues/78)); an unscoped forms/date browse reads the quarterly EDGAR full-index (`master.idx`, 1993 QTR1 onward) via `serveFullIndexBrowse`. Both honor amendment-aware form matching (`10-K` matches `10-K/A`) and cap their scan (`ARCHIVE_PAGE_SCAN_CAP` = 10 pages, `FULL_INDEX_QUARTER_SCAN_CAP` = 8 quarters), disclosing truncation via `total_is_exact` and `dataset.truncated`. ([#77](https://github.com/cyanheads/secedgar-mcp-server/issues/77))
- **`EdgarApiService.fetchFullIndexQuarter`** fetches, parses, and caches (TTL = `EDGAR_TICKER_CACHE_TTL`) a quarterly `master.idx` manifest from `https://www.sec.gov/Archives/edgar/full-index/{year}/QTR{quarter}/master.idx`. The exported `parseMasterIndex` parses the pipe-delimited rows (zero-padding the CIK, deriving the accession number from the filename), and `quartersInRange` enumerates the calendar quarters spanning a date range, newest-first. ([#77](https://github.com/cyanheads/secedgar-mcp-server/issues/77))
- **Result provenance** — every `secedgar_search_filings` row now carries a `source` field (`efts` / `submissions` / `full-index`) identifying which EDGAR backend served it, preserved as a `source` column in the `df_<id>` canvas dataframe. ([#77](https://github.com/cyanheads/secedgar-mcp-server/issues/77))
- **Three new declared error reasons** on `secedgar_search_filings` (`ValidationError`, -32007): `straddling_date_range` (a date range crossing 2001-01-01 — split at the boundary and search each era separately), `pre2001_full_text_unscoped` (free-text terms on a pre-2001 range with no entity scope — no pre-2001 full-text index exists), and `pre2001_full_text_scoped` (free-text terms alongside entity scope on a pre-2001 range — local document matching isn't supported yet). ([#77](https://github.com/cyanheads/secedgar-mcp-server/issues/77))

## Changed

- **`total_is_exact` and `format()` exactness** — `total_is_exact` is now `false` when a pre-2001 archive scan hits its page/quarter cap, in addition to the existing 10,000-hit EFTS cap; `format()` labels archive-sourced results "partial — scan cap reached, more may exist" instead of "capped at 10,000", and appends `| source: <backend>` to each result line. ([#77](https://github.com/cyanheads/secedgar-mcp-server/issues/77))
- **Tool and server descriptions** — `secedgar_search_filings`'s description, its `query` input, and the server's `instructions` string now explain the EFTS 2001 floor and pre-2001 archive routing; README, the design doc, and the agent protocol files carry the same update. ([#77](https://github.com/cyanheads/secedgar-mcp-server/issues/77))
