# 0.7.0 — 2026-05-30

## Added

- **`secedgar_get_insider_transactions`** — Form 3/4/5 insider activity for a company, parsed from ownership XML: reporting person, relationship, transaction date, type, shares, price, and post-transaction ownership. Supports `transaction_type` filter (`purchase`, `sale`, `all`). ([#18](https://github.com/cyanheads/secedgar-mcp-server/issues/18))
- **`secedgar_get_institutional_holdings`** — 13F-HR quarterly institutional holdings parsed from the information table: issuer name, CUSIP, market value, shares, put/call, investment discretion, and filing manager name. Supports `quarter` filter (e.g. `"2025-Q4"`). ([#18](https://github.com/cyanheads/secedgar-mcp-server/issues/18))
- **IFRS friendly-name resolution** in `secedgar_get_financials` and `secedgar_search_concepts` — `revenue`, `net_income`, and `assets` now resolve to their `ifrs-full` tag equivalents when `taxonomy: "ifrs-full"` is passed; `search_concepts` surfaces these entries instead of returning an empty set. ([#19](https://github.com/cyanheads/secedgar-mcp-server/issues/19))
- **`ambiguous_company` error** in `secedgar_get_financials` — thrown when `resolveCik` returns multiple matches, with a `matches` list (CIK, name, ticker) so agents can select the correct entity and retry. ([#23](https://github.com/cyanheads/secedgar-mcp-server/issues/23))
- **`system_catalog_access` error contract** declared in `secedgar_dataframe_query`'s `errors[]` array — the bridge-layer deny on DuckDB system catalogs (`information_schema`, `pg_catalog`, `sqlite_master`, `duckdb_*`) now surfaces as a typed, documented reason. ([#22](https://github.com/cyanheads/secedgar-mcp-server/issues/22))

## Fixed

- **Stale `secedgar_compare_metric` references** in `search-concepts.tool.ts`, `concepts.resource.ts`, and `company-analysis.prompt.ts` updated to `secedgar_fetch_frames`. ([#20](https://github.com/cyanheads/secedgar-mcp-server/issues/20))
