# 0.7.3 — 2026-06-01

## Fixed

- **`EDGAR_MIRROR_REFRESH_CRON` crash on startup** — `node-cron ^4.2.1` is now declared in `dependencies` so it ships in the published npm package and Docker image. Previously the framework's `schedulerService` lazily imported `node-cron` (an optional peer of `@cyanheads/mcp-ts-core` not installed transitively), causing an uncaught module-resolution error that crash-looped the server whenever the cron was configured. ([#26](https://github.com/cyanheads/secedgar-mcp-server/issues/26))
- **Graceful scheduling fault** — `schedulerService.schedule()` / `start()` in `setup()` is now wrapped in try/catch. If scheduling fails for any reason, the server logs a warning and continues serving; the mirror still answers reads with live-API fallback and can be refreshed out-of-band via `bun run mirror:refresh`. ([#26](https://github.com/cyanheads/secedgar-mcp-server/issues/26))
