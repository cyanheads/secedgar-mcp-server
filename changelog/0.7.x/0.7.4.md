# 0.7.4 — 2026-06-01

## Fixed

- **`fetch_frames` partial-mirror coverage** — `tryGetFrames` / `mirrorOrLive` now gates on `companyFactsComplete()` (`status === 'complete'`) instead of the point-lookup readiness flag. A partial or mid-(re)sync company-facts store falls back to the live frames API, preventing a tiny CIK slice from being returned as a complete cross-company frame. ([#29](https://github.com/cyanheads/secedgar-mcp-server/issues/29))
- **`companyfacts-sync` interrupted-ingest checkpoint** — the ingester now emits `checkpoint` only on the terminal page after the full archive stream drains; intermediate pages carry `checkpoint: undefined`. An interrupted ingest leaves no completion checkpoint, so the next `refresh` re-streams the full archive instead of HEAD-skipping a partial store as already-synced. ([#33](https://github.com/cyanheads/secedgar-mcp-server/issues/33))

- `vitest` `^4.1.7` → `^4.1.8`
