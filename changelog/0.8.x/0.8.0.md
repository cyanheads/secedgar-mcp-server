# 0.8.0 — 2026-06-01

Three corrections to `get_institutional_holdings` that change output shape and behavior.

## Added

- **`consolidate` input** (default `true`) — when true, sub-lines for the same security (`cusip` + `title_of_class` + `put_call`) are collapsed into one position (shares and value summed), sorted by `market_value_usd` descending. Set `false` for raw filing rows in filing order. ([#30](https://github.com/cyanheads/secedgar-mcp-server/issues/30))
- **`total_positions`** output field — count of distinct consolidated positions (or raw rows when `consolidate: false`). ([#30](https://github.com/cyanheads/secedgar-mcp-server/issues/30))

## Changed

- **`value_in_thousands` → `market_value_usd`** — field renamed and units corrected. SEC Form 13F Information Table Column 4 reports whole dollars effective 2023-01-03 (Release No. 34-95148); earlier filings reported thousands and are now normalized up to whole USD by filing date. `format()` renders the value with a scaled B/M/K suffix plus the exact figure. ([#27](https://github.com/cyanheads/secedgar-mcp-server/issues/27)) ⚠️ Breaking
- **`quarter` parameter now targets the actual filing** — matches the requested quarter against the SEC submissions `reportDate` (period-end). No match → `no_filings_found` error naming the quarter (no silent substitution of the latest filing). Malformed quarter → `ValidationError`. The EFTS full-text search path is no longer used for quarter resolution. ([#31](https://github.com/cyanheads/secedgar-mcp-server/issues/31)) ⚠️ Breaking
- **`limit`** now applies after consolidation, so it returns the N largest distinct positions (by `market_value_usd` descending) rather than N raw sub-lines. ([#30](https://github.com/cyanheads/secedgar-mcp-server/issues/30))
