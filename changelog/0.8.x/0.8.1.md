# 0.8.1 — 2026-06-01

## Added

- **`limit` input for `get_financials`** — caps inline `data[]` to the most-recent N periods (newest-first). Full series always registers to the canvas dataframe for SQL access. Format output notes the truncation when active. ([#32](https://github.com/cyanheads/secedgar-mcp-server/issues/32))
- **`.regex()` validators for date and period format fields** — `start_date`/`end_date` on `search_filings` and `period` on `fetch_frames` now emit a `pattern` in JSON Schema so format violations (e.g. `2023/01/01`, `2023Q4`) are caught at the MCP layer, not silently forwarded to EFTS. ([#25](https://github.com/cyanheads/secedgar-mcp-server/issues/25))

## Fixed

- **`search_filings` entity targeting** — `cik:`/`ticker:` tokens now scope the EFTS query server-side via the `ciks` param instead of injecting the entity's current name as a required phrase. Filings made under a former company name on the same CIK (e.g. Facebook-era 10-Ks under Meta's CIK 1326801) are now returned. `total`/`total_is_exact` reflect the exact entity match count from EFTS. ([#35](https://github.com/cyanheads/secedgar-mcp-server/issues/35))
- **`dataframe_query` `register_as` schema** — recorded column schema now reflects the result's real DuckDB column types (BIGINT, DOUBLE, etc.) instead of hardcoding VARCHAR. `dataframe_describe` output is accurate for downstream arithmetic and join decisions. ([#28](https://github.com/cyanheads/secedgar-mcp-server/issues/28))
