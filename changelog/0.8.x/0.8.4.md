# 0.8.4 — 2026-06-04

## Fixed

- **`secedgar_get_filing` `document_not_found` / `no_documents` error data** — error payload now returns `documents.primary`, `documents.exhibits`, and `documents.auxiliary` (matching success-path categorization) instead of a flat `available_documents` array that included every XBRL viewer fragment (`R1.htm`…`Rn.htm`, `_cal/_def/_lab/_pre.xml`, `Show.js`, `report.css`). Recovery hint now names the primary document filename directly. ([#37](https://github.com/cyanheads/secedgar-mcp-server/issues/37))
- **`secedgar_search_filings` `forms` field description** — added note that ownership forms (3, 4, 5) are indexed by reporting person (e.g. `"LEVINSON ARTHUR D"`) and carry no issuer name, transaction code, share count, or price; routes insider-transaction workflows to `secedgar_get_insider_transactions`. ([#38](https://github.com/cyanheads/secedgar-mcp-server/issues/38))
