# 0.8.2 — 2026-06-02

The production Docker image shipped `dist/` only — `docker exec <container> bun run mirror:init` (and `mirror:refresh` / `mirror:verify`) failed because the raw `.ts` scripts and the `@/` path alias they depend on were absent. The runtime stage now copies the four mirror scripts and emits a minimal `tsconfig.json` that maps `@/*` → `./dist/*`, so Bun can resolve the alias against the compiled output. The scripts are also added to `files[]` so the npm tarball and `.mcpb` bundle carry the CLI. Closes [#34](https://github.com/cyanheads/secedgar-mcp-server/issues/34).

## Added

- **Mirror CLI in the production image** — the runtime Docker stage copies `scripts/edgar-mirror-init.ts`, `edgar-mirror-refresh.ts`, `edgar-mirror-verify.ts`, and `_mirror-context.ts` from the build stage; emits a minimal `tsconfig.json` mapping `@/*` → `./dist/*` so Bun resolves the path alias against `./dist/`. Operators can now run `docker exec <container> bun run mirror:<init|refresh|verify>` without an external rebuild or volume copy.
- **Mirror scripts in `files[]`** — the four mirror scripts are included in the npm tarball and `.mcpb` bundle.
